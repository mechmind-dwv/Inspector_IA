# 🌌 .cosmicstyleguide.md — Inspector IA Code & Voice Style Guide

This guide was referenced in `CONTRIBUTING.md` but never existed. It does now.

---

## 1. Python code style

- **Formatter**: `black` (line length 100).
- **Imports**: `isort` (profile `black`).
- **Linter**: `ruff` (config in `pyproject.toml`).
- **Type hints**: required on every public function. Use `mypy --strict` where feasible.
- **Docstrings**: Google style. Always document IRA contributions, side effects, and any randomness seed.
- **Naming**:
  - Modules: `snake_case.py`
  - Classes: `PascalCase`
  - Constants: `SCREAMING_SNAKE_CASE`
  - Pattern IDs: `UPPER_SNAKE_CASE_vMAJOR.MINOR` (e.g. `CRYPTO_HIDING_v1.2`)
- **No `print()`** in library code — use `logging` with a module-level logger.
- **No hard-coded paths**, secrets, or sample data inside library modules.

## 2. Commit messages

Conventional Commits:

```
feat(sfe): add OFFSHORE_LAUNDERING generator with shell-chain depth control
fix(graph): handle disconnected components in centrality computation
docs(status): mark CRYPTO_HIDING generator as ready
```

Allowed types: `feat`, `fix`, `docs`, `refactor`, `test`, `chore`, `perf`, `ci`, `build`.

## 3. Branch names

```
feature/<short-name>      # new functionality
fix/<short-name>          # bug fix
docs/<short-name>         # documentation only
chore/<short-name>        # tooling / housekeeping
```

## 4. Pull-request checklist

Every PR must:

- [ ] Update `docs/STATUS.md` if the change alters a module's status.
- [ ] Include tests for new functionality (`tests/`).
- [ ] Pass `ruff`, `black --check`, `mypy`, `pytest` (enforced by CI).
- [ ] Include an **Ethical Impact** note when touching detection logic,
      scoring or data ingestion (3 lines minimum).

## 5. Voice & tone in user-facing text

The "cosmic" metaphor (Supernova Alert, Black Hole, Stellar…) is fine in
**marketing copy, internal naming, and code-name fields**. It must **not**
replace neutral severity labels in operational UI surfaces where journalists
act on the data:

- ✅ Show: `Risk: High (Stellar Anomaly)`
- ❌ Don't: `Risk: 🌟 STELLAR ANOMALY 🌟`

Goal: distinctive identity without compromising professional credibility
with investigative-journalism partners (ICIJ, OCCRP, etc.).

## 6. Documentation tone

- English is the **default** for code, comments and primary docs.
- Spanish translations of user-facing docs are encouraged and live alongside
  with `.es.md` suffix (e.g. `README.es.md`).
- Be honest about status. Use the status emoji set:
  - ✅ working / ready
  - 🟡 partial / skeleton
  - 🔄 planned / in progress
  - ❌ not started / not implemented
  - ⚠️ caveat / risk

Never write `✅ Full Compliance` unless an actual audit document exists in
`docs/audits/`.
